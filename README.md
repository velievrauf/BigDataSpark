# Анализ больших данных - Лабораторная работа №2 - ETL реализованный с помощью Spark

## Введение

Целью работы была реализация ETL-пайплайна (Extract, Transform, Load) с использованием фреймворка **Apache Spark** на языке Python. В рамках работы данные из сырого источника были преобразованы в аналитическую модель "Звезда" в PostgreSQL, а затем на их основе были сформированы 6 аналитических витрин в колоночной базе данных **ClickHouse**.

## Основные этапы работы

### 1. Подготовка инфраструктуры

Инфраструктура была развернута с помощью Docker Compose и включала три основных сервиса:
- **PostgreSQL**: хранилище для сырых данных и итоговой модели "Звезда".
- **ClickHouse**: база данных для хранения итоговых отчетов (витрин).
- **PySpark (Jupyter Notebook)**: среда для написания и запуска ETL-скриптов.

Для корректной работы Spark были автоматически подключены JDBC-драйверы для PostgreSQL и ClickHouse.

### 2. Импорт "сырых" данных

Как и в первой работе, данные из 10 CSV-файлов были загружены в таблицу `staging_raw_data` в PostgreSQL. Эта таблица послужила начальной точкой для работы Spark.

### 3. Построение модели "Звезда" в PostgreSQL

С помощью первого Spark-джобы (`1_postgres_star.ipynb`) была создана модель данных типа "Звезда":
- **Центральная таблица фактов** `star_fact_sales`.
- **Измерения**: `star_dim_customers`, `star_dim_products`, `star_dim_stores`, `star_dim_suppliers`.

В отличие от "Снежинки", в модели "Звезда" география и категории не выносились в отдельные таблицы, а хранились внутри основных измерений, что упрощает структуру и ускоряет аналитические запросы.

### 4. Создание отчетов в ClickHouse

С помощью второго Spark-джобы (`2_clickhouse_reports.ipynb`) данные из "Звезды" были агрегированы и перенесены в ClickHouse. Было создано 6 витрин данных (Data Marts):
1. **Продажи по продуктам**: анализ выручки и популярности товаров.
2. **Продажи по клиентам**: сегментация покупателей и средний чек.
3. **Продажи по времени**: месячные и годовые тренды.
4. **Эффективность магазинов**: выручка среди городов и стран.
5. **Эффективность поставщиков**: средние цены и объем поставок.
6. **Качество продукции**: корреляция рейтингов и объемов продаж.

### 5. Оптимизация для ClickHouse

При записи отчетов в ClickHouse использовалась настройка 'ENGINE=MergeTree()'. Это необходимо для корректного хранения данных в ClickHouse и обеспечения высокой скорости чтения аналитических отчетов.

## Инструкция по запуску

1. Выполнить команду:
   ```bash
   docker-compose up -d
   ```
2. Открыть Jupyter Notebook по адресу `http://localhost:8888`.
3. Последовательно запустить все ячейки в файле `1_postgres_star.ipynb`, а затем в `2_clickhouse_reports.ipynb`.
4. Проверить созданные витрины в ClickHouse через DBeaver (порт 8123, база `petshop_reports`, пользователь `admin`, пароль `4321`).
